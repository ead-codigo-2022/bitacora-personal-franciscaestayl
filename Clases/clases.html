Clase 3

CSS

En clases vimos el concepto CSS, o Cascading Style Sheets u Hojas de estilos en cascada. CSS es el lenguaje que maneja el diseño y la presentación de las páginas web, el diseño de cómo será la página web al visitarla. Este trabaja en conjunto con HTML. El HTML se encarga del contenido básico de la página, mientras el CSS se encarga de darle estilo a la página web.

Objetivos de la clase:

    • Introducción a CSS
    • ¿Cómo escribir CSS?
    • Recomendaciones de escritura
    • ¿Cómo asociar una hoja de estilos a un documento HTML?
    • Selectores CSS
    • Especificidad y Herencia

Clase 4

Tipografía y Modelo de caja

En esta sesión se introdujo el uso de tipografías, el cómo descargarlas desde la web. Para la construcción de un documento HTML, se vio cómo usar distintos tamaños y espaciados para los textos, la alineación para la construcción de párrafos y el uso de colores en distintos elementos de un documento. Por último, se nos introdujo al concepto de Modelo de caja, que consiste en entender que todo lo que está contenido en una página tiene la expresión de una caja, comprender estas cajas es clave para poder crear diseños con CSS.

Al final de la sesión se desarrolla un ejercicio práctico, aplicando los conceptos abordados en esta clase y en clases anteriores.

Objetivos de la clase:

    • Uso de tipografía para sitio web
    • Tamaño y espaciado de texto
    • Alineación
    • Uso de color
    • Modelo de caja

¿Qué es Framework?

Framework es una colección de diseño y estilos.

Sistemas de diseño

Elaboración de una propuesta que no tiene que ver solamente con la estética visual, sino que es todo un concepto de cómo se dispone una marca, tono musical, movimiento, interacciones en la pantalla, etc. Como se declara el diseño desde esos lugares, los principios de diseño que se abordan.

IMB Design Language - Tiene la declaración de elementos de interfaz, grillas, iconografía, etc.

Material Design – sistema de diseño que propuso Google para construir todos sus productos digitales. Sistema de diseño con movimientos, por capas, profundidades, aspectos sensibles visualmente. Orientado al desarrollo, tiene distintos frameworks.

Framework

Nos entrega diseños para tener una base para diseñar. Ej, Bootstrap. Tiene elementos ya definidos. Elementos que hemos visto hasta ahora y hemos hecho paso a paso, en esta pagina ya estan hechas. Está todo documentado, para tener inmediatamente un resultado en html. Permite avanzar rápido en la implementación, y tiene compatibilidad con la mayoría de las plataformas. Es de código abierto. Trabajan con buena practica de código. Son modulares, permiten utilizar ciertos aspectos y extender sus estilos.

Clase 8

Imágenes + Accesibilidad / Seo

Formatos comunes para web

Jpg, imagen de trama ideales para fotografía

png, común pero no conviene para fotografía, es muy pesada, ideales para logos, colores planos (uniformes).

gif, intercambio de imágenes, tiene baja resolución porque trabaja con solo 256 colores. Se recomienda usar más video para los gifs.

svg, imagen vectorial escalable, formato ideal para iconos, logos. Es código, se incrusta en el html y se puede configurar desde ahí.

Webp, reemplaza jpg porque tiene mejor sistema interno, hace que las imágenes sean más livianas.

Optimizaciones de imágenes

Sistemas de compresión que permiten mantener la foto y quitarle 70 por ciento del peso de la imagen. La velocidad de descarga de nuestras imágenes impacta en la accesibilidad y SEO de nuestros sitios.

Se recomienda escoger el formato de imagen correcto. Utilizar imágenes en un tamaño similar al que será mostrada en pantalla.

Imágenes adaptables

Entregar un archivo de imagen diferente dependiendo de características del dispositivo, utilizando los atributos srcset y sizes.

Se entrega una imagen distinta dependiendo del ancho de la pantalla.

Es importante tener considerado los PPP.

Imágenes SVG

Formato de imagen vectorial que puede ser intervenido como código. Es ideal para iconos y logos. Al ser escalable, es responsiva. Puede ser utilizada dentro de o embebidas como código.

Icomoon – uso de iconos svg

Accesibilidad

Se tiene que pensar en la discapacidad para hacer nuestra pagina web; auditiva, cognitiva, neurológica, física, etcétera.

Tambien estan las situacionales, tales como distintos dispositivos de entrada (interactuar con el teclado en caso de que no funcione el mouse por ej.), es fundamental incorporar esas alternativas en las páginas.

La accesibilidad, diseño responsivo y multidispositivo, y optimización para buscadores se cruzan entre sí.

Principios de accesibilidad:

Que tenga percepción; la información debe ser presentada de modo que se pueda percibir por los usuarios.

Que sea operable; los componentes de la interfaz deben ser operables.

Que sea comprensible; la información y el manejo de interfaz debe ser comprensible.

Debe ser robusto.

“Validador HTML”

Niveles de conformidad, se exige un nivel de conformidad para sitios gubernamentales.

Nivel A; uso de color apropiado, control apropiado como el uso de audio.

Nivel AA; exige niveles de contraste mayores. Debe permitir hacer cambio de tamaño del texto. Texto en lugar de imágenes de texto. el sitio lo debe soportar.

Nivel AAA; nivel de contraste aun mayor. Sonido de fondo bajo o ausente. Presentación visual.

Texto en lugar de imágenes (sin excepciones).

Técnicas para la implementación de accesibilidad

Código; utilización de atributos “alt” en imágenes. Utilizar marcado estándar y valido.

Desde el diseño en accesibilidad, es adecuado utilizar calculadoras de contraste.

SEO

Search Engine Optimization (optimización para motores de búsqueda)

Como funciona un buscador.

El buscador tiene una parte que es de rastreo, navega a través de los enlaces del sitio web, como una tela araña, recorre por todos los enlaces, navegando por la web.

Rastreo – indexación – ranking

SEO es un resultado orgánico. Se genera como un ranking de los enlaces. Esto aparece tras el rastreo de los enlaces por individuos (todos). Mientras mas sea visualizada la página, básicamente mejor ranking tendrá.

Atributo “alt”, esto es para indexar las imágenes.

Etiqueta

Es el nombre que aparece al buscar el sitio en la web.

Se recomienda que tenga un máximo de 70 a 75 caracteres.

Meta descripción; se recomienda que este tenga un máximo de 160 caracteres, se inserta en la cabecera.

Estructuras de títulos

El sitio web debe tener solo un “h1”.

Debe seguir un recorrido, esto genera un esquema según su nivel de profundidad.

El sitemap. es para entregarle a Google un índice de todos los enlaces que contiene mi sitio web. Es otro método de optimización.

URL canónica

Se puede entrar a los sitios web sin el triple w, pero de todas maneras debe haber una ruta canónica, una ruta oficial del sitio web. Introduciendo la descripción completa del html.

Es la versión oficial de la dirección de una página web.

Otro método de optimización, definir el idioma del sitio web.
